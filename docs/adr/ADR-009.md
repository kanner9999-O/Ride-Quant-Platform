# ADR-009: Ordering Mechanism

> **Mức của ADR này:** *architectural decision* — chọn mô hình và ràng buộc. **Protocol/implementation chi tiết được deferred sang Phase 1** (xem §6). ADR này không mô tả cách triển khai cụ thể.

## 1. Context

[Chapter 5 §5.4](../constitution/05-time-model.md) (Locked) yêu cầu 2 mức bảo đảm nhưng **không** chốt cơ chế:
- **Mức 1 — intra-partition determinism:** replay cùng log luôn cho cùng thứ tự, không phụ thuộc physical clock.
- **Mức 2 — cross-context causal correctness:** nếu B được tạo do đã quan sát A thì mọi nơi phải thấy A trước B.

Chapter 5 cũng khóa: physical clock (kể cả có NTP) **không đủ** làm ordering authority giữa các node. Với arbitrage đa sàn — use case cốt lõi của Ride — thứ tự sự kiện giữa 2 sàn quyết định trực tiếp lãi/lỗ.

**OQ-005** giữ mở câu hỏi: cơ chế nào thực hiện 2 mức đó?

## 2. Decision

Giải 2 mức bằng **2 cơ chế tách biệt**, không dùng global clock:

### 2.1 Mức 1 — Per-stream contiguous sequence
Mỗi authoritative event thuộc **đúng một** ordered stream. Trong stream, `sequence` là số nguyên **liên tiếp, tăng nghiêm ngặt** (`next = prev + 1`), do **một** writer authority cấp phát **atomically cùng lúc với append**.

### 2.2 Mức 2 — Explicit causation, KHÔNG global total order
Platform **không tuyên bố** có global total order giữa các stream độc lập. Causal correctness đến từ `causation_refs` tường minh (dạng tập, hỗ trợ multi-source causality).

### 2.3 Authoritative partial order — tách global vs run-local
```
P_global = P_stream ∪ P_causation                      (BẮT BUỘC là DAG)
           trên TOÀN BỘ authoritative event graph

P_run    = induced order của P_global trên apply set    (per Input Contract)
           của một Input Contract
```
**Merge policy tạo deterministic topological order trên `P_run`, KHÔNG phải toàn bộ `P_global`.** Tie-break chỉ áp cho event không bị ordered bởi cả `P_stream` lẫn `P_causation`. Cycle hoặc causation đi ngược sequence trong `P_global` là **integrity violation**.

**`P_run` bảo toàn quan hệ bắc cầu** giữa các vertex in-scope, kể cả khi đường nhân quả đi **qua** vertex external: `A ≺ X ≺ B` trong `P_global` (A, B in-scope; X external) ⟹ `A ≺ B` giữ trong `P_run`.

**External satisfied prerequisite:** cause ngoài apply set không phải vertex của `P_run` — chỉ cần immutable committed/existence proof tại append-time; không cần cursor visibility/`stream_position`; không tham gia frontier completeness; cấm đọc payload.

### 2.4 Input Contract cross-mode
Mỗi Decision run pin một Input Contract **versioned + immutable**, chứa `stream_registry_version` (**exact pin**, không dùng range/constraint) · `included_streams` · `merge_policy` · `frontier_policy`.

**Registry applicability tại append:** `event.stream_ref.registry_version = active_registry_at(append_lifecycle_frontier)` — event không được pin registry tương lai chưa active hay registry đã supersede.

**Causation model (semantic thuộc [Chapter 6 §6.7](../constitution/06-identity-model.md) Locked — ADR này KHÔNG định nghĩa lại):**
- `causation_refs` = **direct domain causal predecessor**; tham gia `P_causation`; **KHÔNG** mặc định là payload/state input.
- `related_event_refs` = reference **không mang causal meaning**; không tham gia `P_causation`.
- **Causal closure là ràng buộc cấp Input Contract/run**, không phải semantic cấp event: mỗi Input Contract khai báo `causal_closure_policy` với `dependency_authority: per_effect_event_contract` — **mỗi effect event dùng chính `event_contract_ref` đã pin của nó** để phân loại `causation_refs` nào là state dependency. Cách này machine-readable và không mơ hồ cardinality khi apply set gồm nhiều event type/nhiều Event Contract version. Closure violation làm **Input Contract invalid**, KHÔNG làm semantic bất biến của event thay đổi.

**Activation boundary semantic:** `append_lifecycle_frontier` = lifecycle frontier committed **ngay trước** append transaction (pre-append). Registry activation event KHÔNG được validate bằng chính registry nó kích hoạt — E dùng registry cũ, registry mới active từ chính E.

**Decision in-flight qua registry transition:** Product Owner đã quyết — **Append-and-Revalidate**, ghi tại [ADR-010 §2.6](./ADR-010.md). ADR-009 chỉ sở hữu **registry/frontier boundary**; Decision lifecycle policy thuộc ADR-010. Khi **so sánh hoặc tái tạo cùng một logical Decision/run** giữa các execution mode, mọi mode phải pin **cùng một Input Contract version**; cấm gắn hồi tố. *(Các strategy/run khác nhau được phép dùng Input Contract khác nhau — đây KHÔNG phải yêu cầu toàn platform dùng chung một contract.)*

### 2.5 Frontier / completeness
Live không được authoritative-apply một prefix mà Replay sau đó còn chèn được event đứng trước nó. Frontier chưa complete → **buffer/defer hoặc fail-safe**. **Cấm** lấy local processing wall clock làm completeness authority; watermark chỉ hợp lệ nếu clock/frontier source được **pin, event hóa, replay được**.

### 2.6 Lifecycle & registry authority
**Genesis Registry định nghĩa đúng MỘT canonical Lifecycle Stream và đúng MỘT canonical Audit Stream.** Cả hai là **protected streams**: `stream_id` ổn định · active trong **mọi** registry version · **KHÔNG được retire** · writer được phép handoff nhưng stream identity không đổi. Lý do protected: lifecycle stream phải tồn tại để ghi chính các boundary; audit stream phải tồn tại để bảo toàn fact khi stream khác mất eligibility — retire chúng sẽ tái tạo đúng vòng bootstrap mà Genesis Registry sinh ra để chặn.

*Ranh giới authority:* **ADR-009** sở hữu *sự tồn tại và lifecycle invariants* của Audit Stream; **[ADR-010 §2.6](./ADR-010.md)** sở hữu *Decision nào được dùng preservation path và guardrail của preservation fact*.

Đúng **một** canonical Lifecycle Stream · **Genesis Registry** là bootstrap root duy nhất · activation/retirement có authoritative boundary trên lifecycle stream · retired stream có `terminal_position` · writer handoff giữ sequence liên tục xuyên registry version · registry version sau Genesis mang `effective_from` trỏ lifecycle event.

**Registry applicability (rule mạnh, khớp Chapter 8 §8.3.1):** mọi authoritative cursor phải pin đúng
```
cursor.stream_registry_version = active_registry_at(cursor.lifecycle_frontier)
```
KHÔNG đủ nếu chỉ "registry đã từng visible trước frontier" — registry cũ tại frontier mới sẽ trộn knowledge boundary mới với registry state cũ. Registry cũ tại frontier mới chỉ được dùng trong **counterfactual simulation**: không phải authoritative replay, không được làm parity evidence.

**Hệ quả (boundary chính xác):** một registry activation làm Input Contract pin registry cũ **không còn hợp lệ** cho Decision có **knowledge cut SAU** activation boundary. Với Decision có **knowledge cut TRƯỚC nhưng append SAU** boundary: áp dụng **Append-and-Revalidate Policy** theo [ADR-010 §2.6](./ADR-010.md) — Decision vẫn được append như historical fact, contract cũ vẫn hợp lệ để chứng minh input cut lịch sử, nhưng Decision **chưa đủ điều kiện đi Risk/Execution** cho tới khi revalidate thành công.

**Phân chia thẩm quyền:** `Registry/frontier boundary` → ADR-009 (tài liệu này) · `Decision lifecycle policy qua transition` → [ADR-010 §2.6](./ADR-010.md).

## 3. Alternatives considered

| Phương án | Lý do loại |
|---|---|
| **Lamport Clock** | Bảo toàn chiều happened-before (A → B thì L(A) < L(B)), nhưng **chiều ngược lại không đúng**: L(A) < L(B) KHÔNG chứng minh A gây ra B — không phân biệt được causality với concurrency. Vẫn phải có explicit references để biết dependency thật. |
| **Hybrid Logical Clock (HLC)** | Hữu ích cho temporal locality, observability, distributed timestamping — nhưng **không thay thế được domain causation graph**: clock timestamp không mang đủ domain evidence để biết event nào thực sự là input của quyết định nào. Tăng operational complexity chưa cần ở scale hiện tại. |
| **Global total order** (single global sequencer) | Tạo bottleneck và single point of failure cho toàn platform; partial order + causation graph đã đủ cho mọi yêu cầu đã biết. |
| **Sequence "strictly increasing, cho phép gap"** | Mất khả năng phát hiện event thiếu bằng phép kiểm rẻ nhất (`next != prev+1`). Trong hệ giao dịch, một event thiếu có thể là một fill bị mất → ảnh hưởng Position Ledger. Sẽ cần thêm checksum chain riêng. |
| **Merge sort theo `recorded_time`** | Vi phạm Chapter 5 (Locked): physical clock không phải primary cross-stream ordering key; có thể đặt effect trước cause khi clock skew. |
| **Lifecycle frontier dạng vector (nhiều control stream)** | So sánh `boundary.sequence ≤ frontier.sequence` chỉ có nghĩa khi cùng logical stream; nhiều control stream tạo partial order đa stream cho control plane, phức tạp không tương xứng. |

## 4. Concerns / Risks noted

- **Contiguous sequence ràng buộc implementation:** sequence phải cấp atomically cùng append; cấm reserve-then-crash (tạo lỗ hổng vĩnh viễn). Transaction abort không được tiêu sequence; compaction/restore không được đổi sequence. Đánh đổi này được **ghi nhận trong Draft**; chỉ coi là accepted khi Product Owner approve ADR này.
- **Không có global total order:** khi cần interleave nhiều stream phải khai báo merge policy deterministic trong Input Contract. Được **ghi nhận trong Draft** vì đây là thứ *phải* khai báo tường minh, không nên có default ngầm; accepted khi PO approve.
- **Buffer/defer khi frontier chưa complete** làm tăng latency trong Live. Giới hạn buffer và hành vi overflow → Phase 1 quyết dựa trên đo đạc thật.

## 6. Consequences

**Lamport/HLC về sau:** có thể được bổ sung làm **metadata phụ** (temporal locality, observability, debugging) nhưng **KHÔNG được trở thành nguồn domain causation authoritative** nếu không có ADR mới.

**Ràng buộc lên implementation:** writer authority phải là single-writer per stream với allocation atomic; cần cơ chế phát hiện cycle trước khi append/import; consumer phải resolve causal prerequisite trước khi authoritative-apply.

**Deferred sang Phase 1** (thiết kế/triển khai, không sửa ngầm ADR này — phát sinh quyết định kiến trúc mới thì mở follow-up ADR):
- Chọn watermark vs committed frontier vs barrier event; cách xác định multi-stream completeness.
- Late-arrival protocol; buffer size và overflow behavior.
- Coordinator/checkpoint design.
- Thuật toán topological merge cụ thể; cycle-detection implementation và phạm vi kiểm tra.
- Writer handoff protocol; retirement transaction protocol; registry activation workflow.
- Storage, archive, retention policy và operational recovery.

**Phase 1 deliverable gate (per-capability, KHÔNG chặn toàn bộ Phase 1):** implementation của **component/flow phụ thuộc trực tiếp** vào ordering/frontier/lifecycle semantics không được bắt đầu trước khi design specification **tương ứng** đã qua review. Gate áp theo từng capability, không yêu cầu mọi deferred item hoàn thành cùng lúc:

```
frontier consumer implementation   → cần frontier design
writer handoff implementation      → cần handoff design
retirement implementation          → cần retirement design
archive/retention implementation   → cần retention/archive design
```

**Các hạng mục độc lập KHÔNG bị gate này chặn** (ví dụ domain modeling, schema tooling, module skeleton, những phần không chạm ordering/frontier/lifecycle). Nếu Phase 1 phát sinh lựa chọn làm **thay đổi semantic đã khóa trong ADR này** (ordering model · partial order · Input Contract semantics · frontier invariants · lifecycle authority) → bắt buộc mở **follow-up ADR**, không sửa ngầm ADR này.

**Boundary conformance fixtures** (review/test artifact — KHÔNG phải authority mới; Phase 1 phải thỏa):

| Case | Expected |
|---|---|
| Registry activation `E@n` | E dùng registry active tại `n-1`; registry mới active từ `n` |
| Lifecycle writer transition | old writer ghi boundary; new writer bắt đầu `n+1` |
| Decision cut v3 → registry v4 → append | Decision **stored**; Risk **blocked**; bắt buộc có revalidation result |
| Revalidation **fail** | ghi rejection/supersession event; **không** phát sinh execution intent |
| Revalidation **pass** | execution path được tiếp tục; **phải tham chiếu** revalidation thành công |
| State dependency nằm ngoài closure đã khai báo | **Input Contract invalid** |
| Domain cause KHÔNG phải state input | vẫn giữ trong `causation_refs` + `P_causation` |
| Decision target stream **vẫn active** sau transition | append + block Risk/Execution + revalidate |
| Decision target stream **bị retire** tại transition | preservation fact vào canonical Audit Stream; **không** append trái terminal boundary; không có execution eligibility |
| Revalidation dưới v4, sau đó v5 activate trước Risk | success v4 **stale** → block, revalidate lại |
| Non-state domain cause ngoài input universe | verify identity/existence; không đọc payload; **không** giả lập `stream_position` |
| Cursor pin stale/future registry | **reject** |
| Domain-causal predecessor KHÔNG phải state input | vẫn giữ trong `causation_refs` + `P_causation` |
| State dependency ngoài Input Contract closure | **Input Contract invalid** |
| Revalidation success của Decision A | **KHÔNG** mở execution eligibility cho Decision B |
| Revalidation dưới registry v4 | **KHÔNG** dùng để approve flow yêu cầu registry v5 |

**Quan hệ tài liệu:** [Chapter 8](../constitution/08-event-model.md) khóa *what must always be true*; ADR này khóa *architectural model*; Phase 1 design spec giải *how it is implemented*.
